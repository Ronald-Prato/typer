# Typer DiceBear Avatar

Use this skill for any Typer task that touches user avatars, profile pills,
profile edit, welcome onboarding avatar selection, match/player avatars,
friend/social avatars, bot identity snapshots, or avatar fields in Convex.

## Core Contract

- Typer avatars are DiceBear avatars. Do not replace them with hand-written SVG,
  CSS-only faces, initials-only badges, random images, or a different avatar
  library.
- The installed packages are `@dicebear/core` and `@dicebear/collection`.
  Prefer `createAvatar(avataaars, { seed })` when rendering without depending on
  the remote DiceBear CDN.
- Keep persisted avatar fields compatible with the existing contract:
  `avatarSeed` is the source of deterministic identity, and `avatarUrl` is a
  DiceBear URL generated by `avatarUrlFromSeed()`.
- Public DTOs may expose `avatarSeed` and `avatarUrl`, but must not expose raw
  user-provided SVG or arbitrary image URLs.
- If a remote DiceBear URL is slow, blocked, or visually blank, fix the DiceBear
  rendering path. Do not invent a local non-DiceBear avatar.

## Implementation Rules

1. Express deterministic avatar URL/data-URI rules in `src/domain/avatar.ts`.
2. Keep `UserAvatarImage` mostly presentational: resolve the safe DiceBear
   source, render an image, and fall back to initials only after there is no safe
   avatar source or the image errors.
3. Use the DiceBear `avataaars` collection unless the user explicitly asks for a
   different DiceBear style.
4. Preserve security: reject non-DiceBear stored URLs and sanitize seeds before
   persistence.
5. For bot and match snapshots, copy `avatarSeed` and `avatarUrl` onto the game
   or history document. Do not mutate shared bot user rows to mimic a match.

## Tests And Verification

- Update `src/domain/avatar.test.ts` when changing seed normalization, URL
  generation, DiceBear data URI generation, or URL allowlisting.
- Update `src/components/Avatar/UserAvatarImage.test.tsx` when changing render
  behavior or fallback behavior.
- Use focused checks first:

```bash
pnpm test src/domain/avatar.test.ts src/components/Avatar/UserAvatarImage.test.tsx
pnpm typecheck
```

Do not start the app or dev server unless the current user request explicitly
asks for it.

## Anti-Patterns

- Replacing DiceBear with a custom SVG generator.
- Treating a network failure from `api.dicebear.com` as permission to change the
  avatar system.
- Rendering an empty dark circle as a successful avatar state.
- Storing raw SVG, `javascript:` URLs, or non-DiceBear remote image URLs from
  user input.
